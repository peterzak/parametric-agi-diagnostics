## What This Is

The Distributed Diagnostics Program is a two-phase research program that stress tests AI physical grounding failures using structured diagnostic protocols. It positions human embodied cognition as the primary diagnostic instrument for a class of structural failures in current transformer architectures collectively described as the **Inversion Error**.

Current multimodal AI systems exhibit three reproducible, diagnosable failure modes:

- **Continuity:** The absence of a stable 3D world model, producing spatially incoherent outputs
- **Gravity:** The lack of embodied physical constraint at the moment of generation, producing physically impossible outputs rendered with complete fluency
- **Reversibility:** The absence of a time-reversible operational process, producing a one-way reasoning engine that cannot recover from compounding errors

These failures are invisible from within the systems that produce them and visible from outside them. The Distributed Diagnostics Program provides the instruments to make them empirically measurable.

## The Two Protocols

### Phase 1: The Spaghetti Table Protocol (Screen-Based)
A 15-minute screen-based diagnostic that any researcher can administer to any multimodal AI system using only a browser. No infrastructure required.

**→ [Full Protocol Specification](protocols/spaghetti-table-protocol-v1.md)**

### Phase 2: The Universe Falling Apart Protocol (Embodied XR)
A programmatic proposal for an embodied XR stress testing environment — the Unstable Atrium — in which an Embodied Reviewer inhabits an AI-generated architectural space and experiences physical grounding failures directly. Requires XR infrastructure and institutional partnership.

*Full specification in Paper 2 (see above).*

I am inviting design researchers, HCI practitioners, and design educators to administer the Spaghetti Table Protocol, score their results, and contribute to a shared public dataset.
